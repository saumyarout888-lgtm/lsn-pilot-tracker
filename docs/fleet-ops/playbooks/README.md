# Playbooks — what to open, when

The system-design docs tell you **how the tool works**. These playbooks tell
the 1–3 person team **what to do every day**. They assume the tool from
`docs/fleet-ops/` is already in place.

| Playbook | When to open it |
|---|---|
| [`daily-action-plan.md`](./daily-action-plan.md) | Every morning at 07:45, and for the 10:00 / 13:00 / 17:00 / 20:00 checkpoints |
| [`linkedin-outreach.md`](./linkedin-outreach.md) | Any time you're prospecting a new logistics / e-commerce partner |
| [`conflict-resolution.md`](./conflict-resolution.md) | When a pilot, hub, or client escalates — incl. 3-day no-show and vehicle recovery |

### Rule 0: nothing here is new

Every WhatsApp template name, SQL query, SLA threshold, and ticket category
referenced below already exists elsewhere in `docs/fleet-ops/`. If something
in these playbooks drifts from the source-of-truth docs, **the source-of-truth
docs win** — open a PR to fix the playbook.
